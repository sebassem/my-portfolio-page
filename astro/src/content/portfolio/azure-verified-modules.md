## Project Overview

Azure Verified Modules (AVM) is an initiative to consolidate and set the standards for what a good Infrastructure-as-Code module looks like.

Modules will then align to these standards, across languages (Bicep, Terraform etc.) and will then be classified as AVMs and available from their respective language specific registries.

AVM is a common code base, a toolkit for our Customers, our Partners, and Microsoft. It’s an official, Microsoft driven initiative, with a devolved ownership approach to develop modules, leveraging internal & external communities.

Azure Verified Modules enable and accelerate consistent solution development and delivery of cloud-native or migrated applications and their supporting infrastructure by codifying Microsoft guidance (WAF), with best practice configurations.

### Key Achievements

Authored 6 resource modules for the Azure Verified Modules (AVM) Bicep library, contributing to the standardization of Infrastructure-as-Code across the Azure ecosystem with massive adoption.

- **6 Resource Modules Authored** - Contributing to the AVM Bicep library with modules like:
  - Github and Azure DevOps private runners
  - Deployment scripts
  - Azure Subscription Vending
  - Azure Load testing
  - Azure Privileged Identity Management
- **Massive Adoption** - **~44K+ Azure deployments** to date
- **Quality Standards** - Meeting AVM's rigorous quality requirements

### Self-Hosted Runners Pattern Module

Built a **pattern module for deploying Self-hosted runners** for both GitHub and Azure DevOps. This module serves as:

- **Foundation for Application Landing Zone** - Will be used as the foundation of the future Application Landing Zone workstream
- **CI/CD Enablement** - Enables organizations to run pipelines on their own infrastructure
- **Security & Compliance** - Provides secure, self-managed compute for DevOps
